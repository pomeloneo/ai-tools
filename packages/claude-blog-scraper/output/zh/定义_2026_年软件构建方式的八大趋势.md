# 定义 2026 年软件构建方式的八大趋势

编程 Agent 现在是合作者。

在 2025 年，工程团队发现 AI 可以处理完整的实施工作流：编写测试、调试故障、浏览复杂的代码库。在 2026 年，我们预测这些能力将显著扩展。

我们的新报告确定了我们预测将在今年定义代理式编程 (Agentic Coding) 的八大趋势，分为三类：**基础趋势**（改变开发发生的方式）、**能力趋势**（扩展 Agent 完成的内容）和 **影响趋势**（影响业务成果）。

领先的组织并没有将工程师从循环中移除，而是让工程师的专业知识在最重要的地方发挥作用。

### **我们所看到的**

软件开发生命周期正在经历自图形用户界面以来最重大的变化之一。工程师正在从编写代码转向协调编写代码的 Agent，将他们自己的专业知识集中在架构、系统设计和战略决策上。

通过研究开发人员实际如何使用 AI，出现了一个关键的细微差别：这种转变依赖于积极的协作。

我们社会影响团队的研究表明，虽然开发人员在通过大约 60% 的工作中使用 AI，但他们报告称能够“完全授权”的任务仅占 0-20%。AI 充当着持续的合作者，但要有效地使用它，需要监督、验证和人类判断。

### **实践中的样子**

各行各业的组织正在将这些模式付诸实践，平衡 Agent 的自主性与人类的监督，以便在不牺牲质量的情况下更快地发布。

[乐天 (Rakuten)](https://claude.com/customers/rakuten) 工程师在复杂的任务上测试了 Claude Code：在 vLLM（一个 1250 万行的代码库）中实施激活向量提取方法。Claude Code 在七小时的自主工作中完成了这项工作，达到了 99.9% 的数值准确率。

[TELUS](https://claude.com/customers/telus) 团队创建了 13,000 多个自定义 AI 解决方案，同时将工程代码发布速度提高了 30%，总共节省了超过 500,000 小时。

[Zapier](https://claude.com/customers/zapier) 在整个组织中实现了 89% 的 AI 采用率，并在内部部署了 800 多个 Agent。

### **前进的道路**

对于规划 2026 年优先事项的组织来说，四个领域需要立即关注：掌握多 Agent 协作、通过 AI 自动审查扩展人机监督、将代理式编程扩展到工程团队之外，以及从最早阶段嵌入安全架构。

将代理式编程视为战略重点的组织将定义未来的可能性。
