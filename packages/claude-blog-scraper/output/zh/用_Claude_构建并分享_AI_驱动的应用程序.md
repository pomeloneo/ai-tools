# 用 Claude 构建并分享 AI 驱动的应用程序

**_更新：_** _现已在 Team 和 Enterprise 计划中可用。（2025 年 7 月 31 日）_

**_更新：_** _将 PDF、图像等上传到你的 artifact 中。（2025 年 7 月 17 日）_

今天，我们要介绍在 Claude 应用程序中直接构建、托管和共享交互式 AI 驱动应用程序的能力。现在，开发者可以在他们的 AI 应用程序上更快地迭代，而不必担心因受众增长而带来的扩展复杂性和成本。

### 构建和托管 Claude 驱动的应用程序

这是我们所构建的：Claude 现在可以创建通过 API 与 Claude 交互的 artifacts——将这些 artifacts 转变为 AI 驱动的应用程序，其经济模式实际上非常适合共享。

当有人使用你的 Claude 驱动的应用程序时：

*   他们使用现有的 Claude 帐户进行身份验证
*   他们的 API 使用量计入 _他们自己的_ 订阅，而不是你的
*   你无需为他们的使用付费
*   没有人需要管理 API 密钥

Claude 编写真正的代码来编排复杂的 AI 功能。你可以查看、修改和自由分享它。

### 社区创意

早期用户已经使用交互式 artifacts 构建了：

*   **AI 驱动的游戏**，其中的 NPC 可以记住对话并适应玩家的选择
*   **学习工具**，可根据个人技能水平进行调整并提供个性化辅导
*   **数据分析应用程序**，用户可以上传 CSV 并用自然语言提出后续问题
*   **写作助手**，帮助处理从剧本到技术文档的所有内容
*   **Agent 工作流**，为复杂任务编排多个 Claude 调用

### 开始使用

通过启用这项新的交互功能，开始在 Claude 应用程序中进行构建。只需描述你想创建什么，Claude 就会为你编写代码。

在你们一起工作时，Claude 可以根据你的反馈调试和改进自己的代码。一旦你的应用程序准备就绪，你可以通过链接即时分享——无需部署过程。Claude 处理提示工程、错误处理和编排逻辑等技术细节，让你完全专注于将想法变为现实。

**你可以做什么：**

*   在你的 artifacts 中使用 Claude API
*   处理文件并用 React 创建丰富的 UI
*   查看、分叉 (fork) 和自定义任何 artifact

**当前的限制：**

*   无法调用外部 API（目前）
*   没有持久存储
*   仅限于基于文本的补全 API

此功能目前处于测试阶段，面向 Free、Pro 和 Max 计划用户开放。
