# 构建金融服务 AI Agent

在金融服务领域，AI Agent 正在超越试点项目，提供具体的商业价值。

在银行、财富管理和保险领域，自主 AI Agent 正在改变客户理解消费模式和发现储蓄机会的方式。除了其他用例，这些工具还可以识别潜在的透支费用、建议更好的储蓄策略并指导财务决策。

例如，麦肯锡的研究表明，在欺诈检测中采用 AI Agent 工作流的金融机构可以产生 [200% 到 2000% 的生产力收益](https://www.mckinsey.com/capabilities/risk-and-resilience/our-insights/how-agentic-ai-can-change-the-way-banks-fight-financial-crime)，而挪威银行投资管理公司 (NBIM) 的员工使用 Claude 在分析和运营任务上 [每周累计节省数百小时](https://claude.com/customers/nbim)。

对于大多数组织来说，真正的挑战不是采用 AI Agent。而是构建能够应对复杂法规、管理实时风险并保护客户资产，同时改善业务成果的系统。

## **超越分析：Agent 如何改变游戏规则**

Agent 代表了企业 AI 的根本转变，将依赖持续人工输入和监督的生成式 AI 工具替换为 [自主系统](https://www.anthropic.com/news/claude-for-financial-services)，这些系统以极少（如果有的话）的干预处理长时间运行、上下文繁重的任务。

这种演进在金融服务领域尤其受欢迎，因为数据通常存在于彼此不对话的碎片化系统中，使得更难看到客户财务健康状况的全貌。代理式系统可以理解财务背景，从多个不相关的来源摄取信息，处理多种类型的数据（交易记录、市场数据、监管文件），并将所有这些能力应用于在客户现有的财务工作流中采取有意义的行动。

这在实践中是什么样子的？与其让分析师手动从五个不同的系统中提取数据，审查它，然后更新风险评估，不如让一个 Agent 监控这些系统中的交易模式，识别令人担忧的趋势，根据现行法规起草更新的风险建议，并将它们路由给合适的分析师进行批准。Agent 处理协调和分析，而分析师做出最终决定。

从传统 AI 到 AI Agent 的这种转变对金融服务尤为重要，因为它解决了流程完成问题。财务工作流不仅需要信息，还需要在多个系统中采取行动以实际完成交易并保持合规。Agent 可以弥合这些差距。

## **金融领域 AI Agent 的现实成果**

AI 驱动的金融 Agent 已经在客户服务、欺诈检测和劳动力赋能等领域交付了现实成果。

### **变革客户互动**

客户服务运营是一个自然的起点，因为这一领域已被证明是成功的。实施 AI 驱动客户服务的金融机构正在看到可衡量的改进：

*   多语言虚拟助手现在每年处理数亿次互动，服务于不同语言群体的数百万用户。
*   客户服务 Agent 自动化日常任务，如余额查询和换卡，减少等待时间，同时提供 24/7 响应并建议相关产品，将支持互动转化为实际帮助人们管理财务的机会。

例如，[Intuit TurboTax](https://claude.com/customers/intuit) 构建了一个由 Claude 驱动的 AI 财务助手，为数百万客户生成清晰准确的税务解释。代理式实施非常成功，以至于 AI 驱动的体验获得了比上一纳税年度非 Claude 体验更高的客户评分。

### **打击欺诈和金融犯罪**

在欺诈检测和网络犯罪预防方面，AI Agent 擅长发现因数量庞大而被人类分析师遗漏的模式。当你考虑到金融机构目前 [仅捕获约 2%](https://www.mckinsey.com/capabilities/risk-and-resilience/our-insights/how-agentic-ai-can-change-the-way-banks-fight-financial-crime) 的全球金融犯罪时，你就会明白为什么这很重要。

AI Agent 实时监控数百万笔交易，全天候工作，不受影响人类团队的疲劳或认知限制的影响。麦肯锡发现，在金融犯罪检测工作流中，[一名团队成员](https://www.mckinsey.com/capabilities/risk-and-resilience/our-insights/how-agentic-ai-can-change-the-way-banks-fight-financial-crime) 可以有效监督 20 多个 AI Agent。

对于现代金融平台 [Brex](https://claude.com/customers/brex)，Claude 驱动其 AI 异常检测，审查 100% 的交易，并通过主动分组相关费用、标记政策问题并提供带有建议行动的解释，为金融专业人士提供关键掩护。

### **增强团队能力**

AI Agent 在你的团队最需要的地方提供切实的好处。如果在实施时深思熟虑，这些工具不会取代你的劳动力，而是放大他们所能完成的工作，让他们专注于需要其知识和专业技能的高价值工作。以下是像 [Block](https://www.claude.com/customers/block) 和 [Campfire](https://claude.com/customers/campfire) 这样的金融服务组织是如何做到的：

*   设计团队将想法转化为工作原型，没有编码障碍
*   运营团队自动化案例工单关闭
*   会计团队通过自然语言查询财务数据，执行变动分析，并获得审计支持

## **金融服务的独特挑战**

银行和金融机构面临着独特的挑战，这使得 AI Agent 的实施比典型的企业部署更加复杂。当每一个决定都可能影响客户财务、监管合规和机构风险时，赌注完全处于不同的水平。

复杂的财务背景、监管要求和对客户结果的直接影响相结合，创造了一个彻底性胜过速度的实施环境。以下是你可以预期遇到的一些挑战。

### **与遗留基础设施集成**

金融机构通常运行在数十年前的核心银行系统上，这些系统并非为实时 AI 集成而设计。你的贷款发放系统、交易平台和合规数据库通常使用不同的协议、数据格式和安全模型。

遗留系统集成经常出现在：

*   跨供应商的核心银行平台不兼容
*   需要跨系统编排的部门数据孤岛
*   遗留大型机集成挑战
*   对时间敏感的交易决策的实时同步需求

在应对这些挑战时，团队需要就集成方法做出实际决定。第一个考虑涉及连接性：AI Agent 是否具有与必要系统的直接集成能力？如果没有，团队面临两个实际选择：构建自定义连接器（通常通过 API 或 MCP 方法）或实施中间件系统来弥合这些通信差距。

对于早期的代理式解决方案，寻求与具有 API 和标准协议的现代平台集成。如果你决定需要连接到遗留系统，你需要开发能够在保持交易完整性和审计跟踪的同时在这些系统之间进行转换的中间件。

### **应对监管复杂性**

单笔交易可能会触发来自多个监管机构的合规要求，包括 SEC、FDIC、州银行当局以及负责跨境支付的国际机构。AI Agent 不仅必须了解采取什么行动，还必须了解适用于每个决定的监管框架，以及如何为不同的审计要求记录行动。

需要成为你的 Agent 架构一部分的一些监管考虑因素包括：

*   AI 数据处理工作流的 SOC 2 和 PCI DSS 合规性
*   基于证据的风险评估准确性验证
*   AI 决策审计跟踪的文档要求

确保你从第一天起就在代理式解决方案中构建可观测性和可追溯性。你不仅从故障排除的角度需要它，而且从监管的角度你也绝对需要它。

### **管理实时风险**

与其他可以在稍后审查决策的行业不同，金融 Agent 通常做出的选择会立即影响客户账户、市场头寸或监管地位。这要求具备故障安全架构，Agent 可以快速行动，但始终处于保护客户和机构的预定义风险参数内。

实施需要：

*   金融专业人士可以验证并向客户解释的透明推理
*   针对复杂或模糊财务情况的清晰升级路径
*   允许顾问在客户情况需要时拒绝 AI 建议的覆盖能力
*   优先考虑客户保护而非运营效率的故障安全默认值

确定哪些行动需要人在环 (human-in-the-loop) 授权，无论是从风险还是监管角度。对于高风险行动，考虑系统（代理式或其他）如何在已知的安全状态下失败。

## **实用的部署策略**

AI Agent 已经在改变一些公司的金融运营。目前有大量的 Agent 在生产中运行，并对欺诈检测、客户满意度和运营效率产生了可衡量的影响。那么，你如何构建和部署解决你特定运营挑战的 Agent，同时确保它们符合监管要求和风险管理标准呢？

### **找到你的起点**

最好的 Agent 计划始于针对每个人都已经同意需要修复的事情。清晰的指标在这里起着决定性作用，因为它们显示了解决方案是否真的有效，并有助于为更广泛的采用建立动力。

#### **选择高影响力、低风险的目标**

将组织内添加 Agent 的机会对准那些风险可控但潜在影响有意义的领域。

寻找已经存在人工监督或不完美自动化的后果仍然极小的流程：这些非常适合早期采用，而不会引入过多的组织风险。客户服务分流、内部知识检索和常规数据验证是自然的切入点，AI Agent 可以立即减少工作量，而人类仍保留在验证循环中。

除了直接的生产力收益外，这些初始 Agent 部署是非常有价值的学习经验。每一个低风险的实施都有助于你的团队在没有关键任务截止日期压力的情况下，发展对 Agent 能力、限制模式和集成要求的实际理解。你的技术人员学会在错误是学习机会而不是昂贵失误的环境中微调提示和监控系统。

#### **从简单开始，快速学习**

从处理一项简单任务的 Agent 开始，例如标记异常交易模式、监控合规截止日期或自动化文档分类。

你将获得切实的运营改进，同时让人类判断牢牢保留在循环中，同时建立更雄心勃勃的 Agent 应用所需的组织肌肉记忆。当你最终解决更高风险的用例时，你将拥有通过经验建立的技术能力和信心，而不是关于 Agent 在你特定环境中可能如何表现的理论假设。

### **在整个组织中扩展**

初始实施的成功为企业级能力打开了大门。关键是从单点解决方案转向服务于多个部门的共享基础设施。

#### **构建可重用的基础**

当你构建服务于多个部门的基础 AI Agent 能力，而不是为个别问题部署一次性解决方案时，你的组织将获得更好的结果。

例如，你可能会实施一种可在整个组织中使用的文档处理能力。自动化 [银行对账](https://www.claude.com/customers/campfire) 和发票处理的同一 AI 系统可以帮助合规团队分析监管文件，并支持各部门提取财务数据。每个部门都贡献了加强核心能力的用例，同时受益于其他团队推动的改进。

#### **赢得利益相关者的信任**

Agent 与你的劳动力和客户都有互动，每个群体对 AI 辅助流程的反应都不同。建立信任与技术部署同样重要。

对于客户，透明度至关重要。明确说明他们何时与 AI Agent 互动与何时与人互动，解释 Agent 能做什么和不能做什么，并在需要时提供通往人类专家的直接途径。这种清晰度建立了信心，并鼓励更广泛地采用 AI 辅助服务。

内部采用遵循类似的原则。你的组织已经拥有新系统的变更管理流程。在这里应用它们。员工需要了解 Agent 是如何工作的，何时信任它们的建议，以及如何升级问题。

将对话框架设定为增强而不是替代。例如，[Block 的内部 AI Agent](https://www.claude.com/customers/block) 覆盖了 15 个不同工作岗位（销售、设计、产品、客户成功和运营）的 10,000 名员工中的 4,000 名活跃用户。采用率在一个月内翻了一番，随着员工发现使用它的新方法，用户参与度每周增加 40-50%。

最成功的实施强调 AI 如何增强人类能力，而不是取代它们。

#### **推进到复杂用例**

在这个阶段，你的组织拥有了一支增强的劳动力、完善的核心能力、展示的胜利和准备好迎接更大挑战的经验丰富的团队。从早期实施中吸取的教训为更复杂的 Agent 部署提供了基础。

随着复杂性的增加，你为更简单用例构建的可观测性和人在环机制变得更加关键。你的实施比以往任何时候都更需要：

*   **全面的审计跟踪**，跟踪每个 Agent 决策和使用的数据源，以支持审查和监管合规
*   **实时监控系统**，检测 Agent 何时遇到需要人类判断的边缘情况或不确定场景
*   **升级协议**，根据明确的标准自动将复杂案例路由给适当的专家
*   **绩效指标**，不仅衡量技术准确性，还衡量业务成果和工作流集成成功

## **采取行动**

AI Agent 代表了解决金融服务中长期挑战的重大机遇。成功需要深思熟虑的实施，平衡技术能力与特定行业的要求。这种方法在建立更复杂计划的基础的同时，提供了建立信心的快速胜利。

前进的道路需要技术团队和业务团队之间的合作。那些通过稳健的测试和升级路径优先考虑客户保护，并构建随 AI 能力进步而演进的模块化系统的金融服务领导者，将引领潮流。
