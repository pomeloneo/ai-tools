# Claude 与 Slack

我们推出了两种在 Slack 中使用 Claude 的方式。直接将 Claude 添加到你的 Slack 工作区，或者将 Slack 连接到 Claude 应用程序，这样 Claude 就可以在对话期间搜索并引用相关的 Slack 消息。

随着 Claude 和 Slack 的深度集成，你可以在任何你喜欢的地方工作。直接在 Slack 对话中获得 Claude 的帮助，或者在需要更深入研究项目的上下文时让 Claude 搜索你的 Slack 工作区。起草关键信息的回复，通过提取相关讨论为会议做准备，或分析共享文档——所有这些都在你已经在工作的地方进行。

### 将 Claude 带入 Slack

当你在 Slack 中与 Claude 一起工作时，你会获得你已经使用的所有功能——网络搜索、文档分析和你现有的连接。这意味着 Claude 可以直接在 Slack 中从你的文档、电子邮件和你连接的任何其他工具中提取上下文。

Claude 仅搜索你有权访问的公共和私有 Slack 频道。当你要求 Claude 在公共线程中提供帮助时，它会首先私下起草回复。你可以在与团队分享之前查看、编辑或重新生成，在整个过程中保持控制。

在 Slack 中使用 Claude 有三种方式：

*   **直接消息 (Direct messages)**：与 @Claude 开始私人对话，以获得一对一的研究、写作或分析帮助。
*   **AI 助手面板 (AI assistant panel)**：在任何频道中单击 Slack 的 AI 助手标题中的 Claude 图标，即可访问 Claude，而不会中断正在进行的对话。
*   **线程参与 (Thread participation)**：通过提及 @Claude 将 Claude 添加到任何线程中。Claude 将根据对话上下文起草回复，你可以在分享之前进行审查。

### 将 Slack 连接到 Claude

当你将 Slack 连接到 Claude 时，Claude 会搜索你工作区的频道、直接消息和共享文件，以找到你需要的上下文，从而获得更好的回复和研究。

Claude 通过提取相关的 Slack 讨论、从多个频道收集项目更新或在团队对话中查找隐藏的行动项来帮助你为会议做准备。将其用于：

*   **会议准备：** 审查 Slack 讨论，提取相关文档，并使用最新更新和关键决策创建简报
*   **项目协调：** 跨频道搜索以收集状态更新，识别阻碍因素并总结进度
*   **入职支持：** 通过审查频道历史记录、共享文档和相关电子邮件线程，帮助新团队成员了解正在进行的项目
*   **创建文档：** 将团队讨论形式化为文档，从对话和相关背景材料中提取上下文

这种集成代表了向更智能、AI 驱动的工作流程迈出的重要一步：

“每家公司都在成为代理式企业的道路上，AI Agent 与人类携手工作。与 Anthropic 合作将 Claude 集成到 Slack 中，并将 Slack 上下文集成到 Claude 中，加速了这一旅程——将一流的 AI 直接带入工作流中，”Salesforce 的 Slack 首席产品官 Rob Seaman 分享道。“通过弥合 Claude 和 Slack 用户之间的差距，我们正在创造一种无缝的 AI 体验，使工作比以往任何时候都更高效、更愉快、更智能。”

### 安全与控制

Slack 中的 Claude 维持与用户 Claude 账户相同的安全标准和隐私设置。工作区管理员控制全组织的应用程序批准，个人用户使用其现有的 Claude 账户进行身份验证。

在 Slack 中，Claude 仅访问你有权查看的频道和对话——遵守所有现有的 Slack 权限。Slack 对话遵循你组织的保留政策和安全设置。

### 开始使用

Slack 中的 Claude 今天已通过 Slack 市场向付费 Slack 计划的团队提供。工作区管理员必须先批准 Claude 应用程序，个人用户才能使用其 Claude 账户进行身份验证。

Slack 连接器可供已在 Slack 应用程序中安装 Claude 的 Claude Team 和 Enterprise 计划客户使用。管理员必须先为其组织启用[连接器](https://claude.ai/directory/597f662f-36de-437e-836e-5a81013cbfbe)，然后用户才能在其 Claude 设置的“Connectors”选项卡中进行设置。
