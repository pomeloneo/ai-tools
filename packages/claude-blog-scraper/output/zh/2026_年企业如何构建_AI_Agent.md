# 2026 年企业如何构建 AI Agent

AI Agent 已经从实验性质转变为企业技术栈中不可或缺的一部分。但实际上企业是如何使用它们的呢？

我们与研究公司 Material 合作，调查了各行各业和不同公司规模的 500 多名技术领导者，以了解组织目前如何部署 Agent，以及他们看到了哪些未来的机会。

调查结果显示了一个清晰的模式：组织正在从简单的任务自动化转向跨团队和业务职能的复杂、多步骤工作流。

## **数据说明了什么**

超过一半的组织 (57%) 现在部署了用于多阶段工作流的 Agent，其中 16% 运行着跨多个团队的跨职能流程。到 2026 年，81% 的组织计划处理更复杂的用例，其中 39% 将开发用于多步骤流程的 Agent，29% 将部署用于跨职能项目的 Agent。

编程领域引领了应用潮流。近 90% 的组织使用 AI 辅助开发，86% 的组织在生产代码中部署了 Agent。组织报告称，整个开发生命周期都节省了时间：规划和构思 (58%)、代码生成 (59%)、文档编写 (59%) 以及代码审查和测试 (59%)。

但影响不仅限于工程领域。数据分析和报告生成 (60%) 以及内部流程自动化 (48%) 位列影响力最大的用例。展望未来，56% 的组织计划在明年实施用于研究和报告的 Agent。

或许最值得注意的是，80% 的组织报告称，他们的 AI Agent 投资已经带来了可衡量的经济回报。

## **实践中的样子**

看到成效的组织正在将 Agent 视为其基础设施的核心部分，而不是实验品。

[汤森路透 (Thomson Reuters)](https://www.claude.com/customers/thomson-reuters) 使用 Claude 为其 AI 法律平台 CoCounsel 提供支持。律师们曾经花费数小时手动搜索文档，现在可以在几分钟内访问 150 年的判例法和 3,000 名领域专家的知识。

网络安全公司 [eSentire](https://www.claude.com/customers/esentire) 将专家威胁分析时间从 5 小时压缩到 7 分钟，AI 驱动的分析与他们的高级安全专家的判断一致率高达 95%。在医疗保健领域，[Doctolib](https://claude.com/customers/doctolib) 在其整个工程团队中推出了 Claude Code，在数小时而不是数周内替换了旧的测试基础设施，并将功能发布速度提高了 40%。

零售行业也看到了类似的收益。[欧莱雅 (L'Oréal)](https://claude.com/customers/loreal) 在对话式分析方面达到了 99.9% 的准确率，使 44,000 名月度用户能够直接查询数据，而无需等待定制的仪表板。

## **前进的道路**

对于 2026 年的领导者来说，问题不在于是否采用 AI Agent，而在于如何战略性地扩展它们。数据指出了三个主要挑战：与现有系统的集成 (46%)、数据访问和质量 (42%) 以及变革管理需求 (39%)。

十分之九的领导者报告说，Agent 正在改变他们团队的工作方式，员工将更多时间花在战略活动、建立关系和技能发展上，而不是常规执行上。

这种转变需要专门构建的基础设施：针对编程和企业工作流优化的模型、像 Agent SDK 这样的框架，以及像 Claude Code 这样帮助团队更快地从原型走向生产的工具。

我们还发现，虽然编程一直是 AI Agent 的试验场，但这仅仅是个开始。随着 Agent 扩展到研究、客户服务、财务规划和供应链运营，现在建立专业知识的组织将在技术成熟时获得不成比例的价值。
